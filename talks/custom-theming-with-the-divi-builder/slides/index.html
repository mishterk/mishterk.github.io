Custom theme dev using the Divi builder plugin

Phil Smart

Why use the custom Divi builder?

  • Keep things quick for us – easy reuse of components via import/export
  • Keep things flexible for the client (nice admin UI)

Likes

  • The interface is quite nice - it’s not very WordPress, but it’s clear and easy to understand. Also very flexible.
  • The ability to save things to the library is a powerful feature, particularly the save as global feature.
  • The reasonable flexible in-admin CSS-editing capabilities are also quite handy.

Don't likes

  • The CSS is a nightmare
  • The code itself is poorly organised – the modules are all written in one document which makes and intelligent IDEs work really hard to inspect the doc
  • The support isn’t very good – seems to be simply a community-driven forum for helping people at a site admin level

Custom Module Registration (some tips)

  • Doesn’t appear to be much helpful information on this one – what I’ve found appears to be mostly incorrect
  • Hook onto the et_builder_ready hook
  • If you update the admin fields, you need to invalidate your local storage entries
  • Check out the demo here

Issues I've faced

  • Overriding the CSS is a pain – the base CSS basically resets anything within the divi builder
  • I did have some issues with custom module registration – a lot of endless white loading screens
  • Hyphens can cause issues in field registration
